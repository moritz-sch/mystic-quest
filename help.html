Mystic Quest Editor Manual

Introduction

This editor allows changing many variables pertaining to enemies, bosses, enemy attacks, spells, items, shops and level-ups in Mystic Quest respectively Final Fantasy Adventure respectively Seiken Densetsu. For the most part the program should be self-explanatory, a few parts are not however. Please read this document before starting to use the editor.

How to run

The program requires Java. In Linux use "java -jar (filename)" to run it. In Windows the Java Runtime Environment will probably automatically associate with jar files.

ROM information

The program works with all versions of the game. When loading a file the editor tries to autodetect which version it is by comparing checksums in the ROM header. If the checksum has been altered it will fail or possibly even suggest a wrong version. In any case the user is asked to confirm.

Things You Should Know

General

If you want to transfer the changes you made in the editor to another ROM, use the "Save to" command. It allows saving from any version to any other. It does not open the file to which you save and does not alter the opened file in any way.

In many cases the values are only base values. For instance the Goblin enemy has a base value of 8 HP, but in-game small random variations occur, so Goblins can have 7 HP or 9 HP.

Most variables can take values from 0 to 255. Exceptions are mentioned below.

Enemies

HP

Must be a multiple of 4 between 4 and 1020. Other values can be entered but when saving the editor will force it to be a valid number.

Speed

The movement speed of enemies. A lower number means a faster speed, except for 0 which is very slow. The slowest moving enemies in the game have a speed of 12, larger values will result in very slow movement speed.

Attack

This is the type of attack an enemy uses, if it uses any. If it is set to "(none)" an enemy won't use any attack. If an attack is set the enemy still may not use it if said enemy is not programmed to use any attacks (for example Rabites have the same attack as Goblins but will never throw knives). Maybe not all attacks will work with all enemies, I have yet to find out.

Immunities

The player attacks are divided into seven groups, for each of which you can select if an enemy takes damage from it or not. The groups are as follows.

  1. Excalibur, Nuke
  2. Thunder Spear, Lightning
  3. Ice Sword
  4. Flame Whip, Fire
  5. Star (Flail), Mattock
  6. Silver Sword
  7. Broad Sword, Blood Swoard, Dragon Sword, Rusty Sword, Battle Axe, Zeus Axe, Sickle, Chain, Wind Spear

The Ice spell is a special case: an enemy is susceptible to it if and only if it is susceptible to both the Ice Sword and Silver Sword.

Status Effects

While this is never used in the original game, it is possible for enemies to cause more than one status effect.

Bosses

HP

Must be a multiple of 16 between 16 and 4080. Other values can be entered but when saving the editor will force it to be a valid number.

Speed

See enemies.

Attacks

See enemies.

Phases

During battle bosses go through various phases, and several variables can be set independently for each phase. To find out which phase belongs to which number in the editor you will have to try it out, I'm afraid.

Status Effects

Bosses can cause status effects, too, another thing which was not used in the original game. Status effects must be set for each phase individually. Like with enemies multiple status effects are possible.

Attacks

Speed

See enemies.

Spells

MP Cost

Must be a value between 0 and 31.

Items

Price

Maximum value is 65535.

Resistances (Shields)

See http://www6.plala.or.jp/fom/seiken/bogu.html for information on which resistance is which. It is in Japanese but a machine translation should be good enough to understand most of it.

Experience

The program does not check if the sequence of values is increasing. You probably want them to be.

Caveats

The GUI is a bit wonky. In some cases if you change a value and save the file the value might not be written to the ROM. Go to another field before saving to make sure that it doesn't happen.

Mousewheel scrolling in some subframes is incredibly slow for some reason. Since you can use the scrollbar or the PgUp and PgDn keys there are easy workarounds. Sorry about that.
